## 核心理念

胜率预测本质是**多因子加权评分系统**。不预测具体涨跌幅，只回答一个问题：**当前时点买入，赢的概率有多大？**

### 设计原则

1. **多维交叉验证** - 单因子会失效，多维度才稳健
2. **量化减少偏见** - 数字说话，不靠感觉（v2.0: 代码自动计算）
3. **可回测可优化** - 框架固定，权重可调
4. **承认局限性** - 历史不代表未来，黑天鹅无法预测
5. **环境自适应** - 宏观环境不同，维度权重自动切换
6. **🆕 工程化落地** (v2.0) — 不再是纯文本规则，有可执行的 Python 引擎

## 🔄 动态权重调整机制（继承自v1.1.0）

> **核心洞察**：牛市中趋势和资金更重要，熊市中估值和情绪（逆向）更重要。

### 宏观环境分类

基于 `detect_macro_environment()` 函数半自动判定（或预置参数）：

| 环境 | 判定信号 | 权重调整方向 |
|------|----------|--------------|
| **牛市/顺风** | 60日涨>15% + 低波动 | 趋势↑↑ 资金↑ 估值↓ 情绪↓ |
| **中性** | 无明确方向 | 维持默认 30/25/25/20 |
| **熊市/逆风** | 60日跌>10% + 高波动 | 估值↑↑ 情绪↑↑ 趋势↓ 资金↓ |
| **危机模式** | 波动率>4% 或深跌>20% | 情绪↑↑↑ 趋势↓↓↓ 资金↓ |

### 权重矩阵（股票·默认）

| 维度 | 牛市 | 中性 | 熊市 | 危机 |
|------|-----|------|------|------|
| **趋势** | 35% | 30% | 20% | 15% |
| **估值** | 20% | 25% | 30% | 25% |
| **资金** | 30% | 25% | 20% | 15% |
| **情绪** | 15% | 20% | 30% | 45% |

### 权重矩阵（加密货币·默认）

| 维度 | 牛市 | 中性 | 熊市 | 危机 |
|------|-----|------|------|------|
| **估值** | 25% | 32% | 35% | 35% |
| **趋势** | 30% | 23% | 18% | 12% |
| **资金** | 28% | 22% | 17% | 13% |
| **情绪** | 17% | 23% | 30% | 40% |

## 🔗 与 financial-tycoon 互操作（v2.0 更新命名）

### 接收 tycoon 输入

```
1. 宏观环境判定：
   - tycoon 步骤① → 宏观研判结论 → 大神选择对应权重模式（--macro_env 参数）
   - 如果 tycoon 未提供，大神独立调用 detect_macro_environment() 自动判定

2. 数据验证状态：
   - tycoon 步骤⓪ → 数据验证通过 → 大神可正常计算
   - tycoon 步骤⓪ → 数据异常 → 大神也暂停，等数据修正

3. 定性分析方向：
   - tycoon 给出买/卖/观望建议 → 大神胜率作为量化验证
```

### 输出给 tycoon

```
1. 胜率评分 → 嵌入 tycoon 步骤⑦（风控计算）
2. 各维度得分 → 帮助 tycoon 识别薄弱环节
3. 定性定量分歧标记 → 触发 tycoon 的深度审查
```

### 分歧处理规则（不变）

| 情况 | Tycoon说 | 大神说 | 处理 |
|------|---------|--------|------|
| 一致看多 | 买入 | ≥55% | ✅ 正常执行 |
| 一致看空 | 回避 | <45% | ✅ 正常执行 |
| **Tycoon多+大神空** | 买入 | <45% | ⚠️ 分歧，减半仓位或观望 |
| **Tycoon空+大神多** | 回避 | ≥70% | ⚠️ 分歧，检查Tycoon是否遗漏因子 |

> 🔴 **分歧 = 信息不完整的信号**，不是选边站的信号

## 🐍 Python 计算引擎（v2.0 核心）

### 引擎文件：`scripts/dashen_engine.py`

**这是 v2.0 最关键的升级——从"说明书"变成真正的工具。**

### 使用方法

```bash
# 基础用法（自动拉数据+打分+输出报告）
python scripts/dashen_engine.py --code 00700.HK --asset_type stock

# 加密货币
python scripts/dashen_engine.py --code BTC --asset_type crypto

# 指定行业（触发行业差异化权重）
python scripts/dashen_engine.py --code 000001.SZ --asset_type stock --industry 银行
python scripts/dashen_engine.py --code 300750.SZ --asset_type stock --industry 科技成长

# 预置宏观环境（跳过自动判定）
python scripts/dashen_engine.py --code 00700.HK --macro_env bull

# 输出到文件
python scripts/dashen_engine.py --code 00700.HK --output report_700.json
```

### 引擎架构

```
dashen_engine.py
├── 数据获取层
│   ├── fetch_stock_data()     → 东方财富 / 新浪 自动切换
│   └── fetch_crypto_data()    → CoinGecko / Binance 自动切换
├── 因子计算层 (DaShenEngine 类)
│   ├── 趋势因子: F1-F4
│   ├── 估值因子: F5-F7
│   ├── 资金因子: F8-F10
│   ├── 情绪因子: F11-F14
│   └── 新增因子: F15-F19 (v2.0)
├── 权重引擎
│   ├── detect_macro_environment()  → 半自动宏观判定
│   └── get_weights()              → 行业×环境 二维权重矩阵
└── 报告输出
    └── JSON 结构化报告 (胜率/置信度/各因子明细/建议)
```

## 胜率计算公式

```
单因子得分范围：[-2.0, +2.0] （v2.0: 11档精度，步长0.5）
维度得分 = Σ(因子归一化得分)      ← 各因子在维度内的权重归一化
总分 = Σ(维度得分 × 动态权重) + 新增因子附加分(≤10%)
标准化胜率 = (总分 + 2) / 4 × 100%
范围: [0%, 100%]
```

### 胜率区间解读

| 胜率区间 | 信号强度 | 建议操作 |
|----------|----------|----------|
| **70%+** | ★★★★★ | 强烈买入，可重仓（≤30%） |
| **55-70%** | ★★★★☆ | 可以建仓，分批买入 |
| **45-55%** | ★★★☆☆ | 观望，等待更明确信号 |
| **30-45%** | ★★☆☆☆ | 谨慎，考虑减仓 |
| **<30%** | ★☆☆☆☆ | 回避，不宜介入 |

### 置信度评估（v2.0 新增）

| 置信度 | 条件 | 含义 |
|--------|------|------|
| **高** | ≥85%因子有非零得分 | 数据充分，结果可靠 |
| **中** | 70-85% | 部分因子缺数据，基本可用 |
| **低** | <70% | 多数因子为占位值，仅供参考 |

## 风险提示

⚠️ **重要声明**：

1. 本系统仅供参考，不构成投资建议
2. 历史规律不代表未来表现
3. 黑天鹅事件（战争、政策突变等）无法预测
4. v2.0 的 Python 引擎依赖公开免费 API，可能存在延迟或不准确
5. F14(VIX)/F15(内部人)/F16(分析师)/F17(做空) 目前为占位因子，需接入专业数据源后生效
